# 在自研 C++ 引擎中构建完整的 UI 系统

*2026年3月 · 8 分钟阅读*

## 引言

当我们 7 人团队着手在自研 ECS 引擎上构建多人潜行游戏时，我担任了 UI/UX 系统负责人。在六个月中，我设计并实现了一个完整的 UI 框架，包含 7,100+ 行 C++ 代码、47 个文件和 15+ 个全屏界面。

本文介绍了使用 Dear ImGui 在自研实体组件系统引擎上从零构建生产级 UI 系统的架构决策、技术挑战和经验教训。

## 挑战

我们的引擎使用严格的 ECS 架构，所有游戏状态都存储在注册表上下文资源中。UI 系统必须：

1. **遵循 ECS 数据流** —— 不允许系统间直接调用
2. **支持 15+ 个不同界面** —— 从启动画面到游戏结算
3. **处理多人游戏状态** —— 大厅、队伍选择、联网 HUD
4. **兼容 PC 和 PS5** —— 鼠标、键盘和手柄
5. **保持视觉一致性** —— 贯穿所有界面

## 架构概览

```
UI 系统 (Sys_UI, 优先级 500)
├── 全屏页面
│   ├── 启动画面 → 主菜单 → 设置
│   ├── 暂停菜单 → 游戏结算
│   ├── 背包 → 任务选择
│   └── 大厅 → 加载
├── HUD（7个独立子模块）
│   ├── 生命/护甲条
│   ├── 小地图（NavMesh → 屏幕空间）
│   ├── 装备栏（武器 + 道具）
│   ├── 罗盘
│   ├── 动作通知（消息队列）
│   ├── 交互提示（3D → 2D 投影）
│   └── 聊天叠层
└── 覆盖系统
    ├── 道具轮盘（径向选择）
    └── 对话面板（方向键输入）
```

关键思路是将每个 UI 模块视为**纯渲染函数**，从 ECS ctx 资源读取数据并绘制到屏幕。模块不存储自己的状态 —— 所有状态都在共享资源中，如 `Res_UIState`、`Res_GameState` 和 `Res_InventoryState`。

## 集中化主题系统

开发早期，我注意到 UI 文件正在积累硬编码颜色和魔法数字。有 30+ 个文件，这将变得不可维护。解决方案是 `UITheme.h`：

```cpp
namespace ECS::UITheme {
    // 5色暖色调色板
    inline const ImVec4 kBgWindow  = {0.961f, 0.933f, 0.910f, 1.0f}; // #F5EEE8
    inline const ImVec4 kAccent    = {0.988f, 0.435f, 0.161f, 1.0f}; // #FC6F29
    inline const ImVec4 kTextPrimary = {0.063f, 0.051f, 0.039f, 1.0f}; // #100D0A
    // ... 加上绘制列表辅助函数
    inline ImU32 Col32_Accent(uint8_t a = 255) { return IM_COL32(252,111,41,a); }
}
```

这个单一头文件成为了整个 UI 的唯一事实来源。更改配色方案只需编辑一个文件。我甚至在个人作品集网站上复用了相同的调色板 —— 一致性证明了该系统的有效性。

## HUD 分解

HUD 是最复杂的子系统。我没有使用一个庞大的渲染函数，而是将其分解为 7 个独立模块：

| 模块 | 读取资源 | 绘制内容 |
|------|---------|---------|
| 生命条 | Res_GameState | HP/护甲条，带比率着色 |
| 小地图 | Res_MinimapState | NavMesh 投影俯视图 |
| 装备栏 | Res_InventoryState | 当前武器 + 道具槽位 |
| 罗盘 | Res_GameState | 基本方向指示器 |
| 动作通知 | Res_ActionNotifyState | Toast 卡片（6条消息队列） |
| 交互提示 | Res_GameState | 3D→2D 投影标签 |
| 聊天叠层 | Res_ChatState | 消息历史覆层 |

每个模块可以独立开发、测试和修改。当队友需要调整小地图时，他们只需修改 `UI_HUD_Minimap.cpp` —— 不会有破坏生命条的风险。

## 经验教训

1. **尽早投资主题系统** —— 随着项目增长，回报倍增
2. **大胆分解** —— 7 个小文件永远优于 1 个大文件
3. **让 ECS 驱动 UI 状态** —— 对抗架构只会制造 Bug
4. **为最差输入设计** —— 如果手柄能用，那到处都能用

## 结语

从零构建 UI 系统是这个项目中最有收获的挑战之一。严格的 ECS 约束不但没有限制我们，反而带来了更清晰的架构。模块化设计使团队能够并行开发，集中化主题确保了临时样式无法实现的视觉一致性。
